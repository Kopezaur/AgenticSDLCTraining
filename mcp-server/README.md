# MCP Server for Task Manager

A Model Context Protocol (MCP) server that allows AI agents to interact with the Task Manager database. The server provides tools for schema inspection, bulk data insertion, and summary statistics.

## Architecture

```
[ AI Agent (Claude, GPT-4o, etc.) ]
              ⇅  MCP protocol (SSE)
[ MCP Server (this project, port 8081) ]
              ⇅  JDBC
[ PostgreSQL (shared with Task Manager backend) ]
```

## Prerequisites

- Java 17+
- Maven
- PostgreSQL with the `taskdb` database (see root README for setup)

## Setup & Run

```bash
cd mcp-server
mvn clean install
mvn spring-boot:run
```

The MCP server starts on `http://localhost:8081`. The SSE endpoint is available at `http://localhost:8081/sse`.

## Available Tools

| Tool | Description |
|------|-------------|
| `mcp-schema-tasks` | Returns the database schema for the tasks table as a simplified JSON Schema |
| `mcp-tasks` | Accepts a list of Task objects and inserts them into the database |
| `mcp-tasks-summary` | Returns summary statistics (total count and breakdown by status) |
| `mcp-help` | Returns a description of all available tools |

## Connecting an AI Agent

### Claude Desktop / Claude Code

Add to your MCP configuration:

```json
{
  "mcpServers": {
    "task-manager": {
      "url": "http://localhost:8081/sse"
    }
  }
}
```

### Example Prompts

**Inspect the schema:**
```
Use the mcp-schema-tasks tool to show me the structure of the tasks table.
```

**Generate and insert 1000 tasks:**
```
Please inspect the task schema using mcp-schema-tasks. Then generate and insert 1000 diverse
tasks with random statuses, titles, and due dates using the mcp-tasks tool. Use realistic
task titles related to software development.
```

**Check the results:**
```
Use mcp-tasks-summary to show me how many tasks are in the database and their status breakdown.
```

**Full workflow:**
```
1. First, call mcp-help to see what tools are available.
2. Then call mcp-schema-tasks to understand the data structure.
3. Generate 1000 realistic software development tasks with varied statuses and due dates,
   and insert them using mcp-tasks.
4. Finally, call mcp-tasks-summary to verify all 1000 tasks were inserted.
```

## Tool Details

### mcp-schema-tasks

**Parameters:** None

**Example response:**
```json
{
  "table": "tasks",
  "columns": {
    "id": { "type": "bigint", "primaryKey": true, "autoGenerated": true },
    "title": { "type": "varchar(100)", "required": true },
    "description": { "type": "varchar(500)", "required": false },
    "status": { "type": "varchar(20)", "enum": ["TODO", "IN_PROGRESS", "DONE"], "default": "TODO" },
    "dueDate": { "type": "date", "required": false, "format": "yyyy-MM-dd" }
  }
}
```

### mcp-tasks

**Parameters:**
- `tasks` (required): List of task objects

**Example input:**
```json
[
  {
    "title": "Implement login page",
    "description": "Create the login form with email and password fields",
    "status": "TODO",
    "dueDate": "2026-05-01"
  },
  {
    "title": "Fix navigation bug",
    "description": "Menu doesn't close on mobile after selecting an item",
    "status": "IN_PROGRESS",
    "dueDate": "2026-04-15"
  }
]
```

**Example response:**
```json
{"inserted": 2, "failed": 0}
```

### mcp-tasks-summary

**Parameters:** None

**Example response:**
```json
{
  "totalTasks": 1005,
  "byStatus": {
    "TODO": 400,
    "IN_PROGRESS": 350,
    "DONE": 255
  }
}
```

### mcp-help

**Parameters:** None

Returns a plain-text description of all available tools and the typical workflow.

## Database Migrations

Flyway migrations are located in `src/main/resources/db/migration/`:

- `V1__create_tasks_table.sql` — Creates the `tasks` table
- `V2__seed_sample_tasks.sql` — Inserts 5 sample tasks for development

Migrations run automatically on startup.
